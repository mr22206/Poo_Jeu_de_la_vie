    Foire Aux Questions (FAQ)¶

    Questions générales¶

    Q: Qu'est-ce que le Jeu de la Vie ?¶

    R: Le Jeu de la Vie est un automate cellulaire créé par John Conway en 1970. Il simule l'évolution de cellules selon des règles simples de naissance, survie et mort.

    Q: Quelles sont les règles de base ?¶

    R: Les règles sont : 1. Une cellule morte avec exactement 3 voisines vivantes naît 2. Une cellule vivante avec 2 ou 3 voisines vivantes survit 3. Dans tous les autres cas, la cellule meurt ou reste morte

    Installation¶

    Q: Comment installer sur Linux ?¶

    R: Suivez ces étapes : 

    # Installation des dépendances
sudo apt install build-essential cmake libsfml-dev

# Compilation
git clone https://github.com/votre-repo/jeu-de-vie.git
cd jeu-de-vie
mkdir build && cd build
cmake ..
make

    Q: L'installation échoue, que faire ?¶

    R: Vérifiez : 1. Les dépendances sont correctement installées 2. La version de CMake (3.10+ requise) 3. Les logs d'erreur dans build/CMakeFiles/CMakeError.log

    Utilisation¶

    Q: Comment charger un motif personnalisé ?¶

    R: Deux méthodes : 1. Via l'interface graphique : Menu → Charger → Sélectionner fichier 2. Via la ligne de commande : ./jeu-de-vie -p motif.gol

    Q: Comment ajuster la vitesse de simulation ?¶

    R: - Mode graphique : Touches +/- ou molette de la souris - Mode console : Options -s <vitesse> au lancement

    Problèmes courants¶

    Q: Le jeu est lent, comment l'optimiser ?¶

    R: Essayez : 1. Réduire la taille de la grille 2. Désactiver les effets graphiques 3. Utiliser le mode console pour les grandes simulations 4. Compiler en mode Release

    Q: Les motifs ne s'affichent pas correctement ?¶

    R: Vérifiez : 1. Le format du fichier de motif 2. La taille de la grille 3. La position initiale du motif 4. Les paramètres d'affichage

    Développement¶

    Q: Comment contribuer au projet ?¶

    R: Vous pouvez : 1. Fork le dépôt 2. Créer une branche pour votre fonctionnalité 3. Suivre les guides de contribution 4. Soumettre une Pull Request

    Q: Comment ajouter un nouveau renderer ?¶

    R: 1. Créer une classe héritant de IRenderer 2. Implémenter les méthodes requises 3. Ajouter les tests unitaires 4. Mettre à jour la factory de renderers

    Performance¶

    Q: Quelle est la taille maximale recommandée ?¶

    R: - Mode graphique : 1000x1000 cellules - Mode console : 5000x5000 cellules - Mode debug : Dépend de la mémoire disponible

    Q: Comment améliorer les performances ?¶

    R: 1. Utiliser la compilation optimisée 2. Activer le multithreading 3. Ajuster les paramètres de rendu 4. Utiliser le mode approprié

    Sauvegarde et chargement¶

    Q: Où sont stockées les sauvegardes ?¶

    R: Par défaut dans : - Linux : ~/.local/share/jeu-de-vie/saves/ - Windows : %APPDATA%\jeu-de-vie\saves\ - macOS : ~/Library/Application Support/jeu-de-vie/saves/

    Q: Quel format de fichier est utilisé ?¶

    R: Les fichiers utilisent : - .gol pour les motifs - .save pour les sauvegardes de partie - .conf pour la configuration
