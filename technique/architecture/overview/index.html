    Architecture du Projet¶

    Vue d'ensemble¶

    Principes architecturaux¶

    • Modularité : Composants faiblement couplés
    • Extensibilité : Points d'extension clairs
    • Testabilité : Composants isolés facilement testables
    • Maintenabilité : Code organisé et documenté

    Pattern Observer¶

    • Subject (Game)
    • Maintient liste d'observateurs
    • Notifie lors des changements d'état

    • Méthodes attach(), detach(), notify()

    • Observers

    • IRenderer : Met à jour l'affichage
    • FileHandler : Sauvegarde l'état
    • Réagissent via update(Subject*)

    // Exemple d'implémentation
class Game : public Subject {
    void update() {
        grid.evolve();
        notify(); // Notifie tous les observateurs
    }
};

class IRenderer : public Observer {
    void update(Subject* subject) override {
        render(static_cast<Game*>(subject)->getGrid());
    }
};

    Flux de données¶

    Cycle de vie d'une frame¶

    1. Input utilisateur → Controller
    2. Controller → Model (mise à jour)
    3. Model → View (notification)
    4. View → Rendu écran

class Game {
    void update() {
        // 1. Collecte des inputs
        InputState inputs = inputHandler.getState();

        // 2. Mise à jour du modèle
        grid.evolve();

        // 3. Notification de la vue
        renderer->draw(grid);
    }
};

    • Utilisation de smart pointers
