    Extensibilité

    Vue d'ensemble

    Le projet est conçu pour être facilement extensible via plusieurs points d'entrée. Cette documentation explique comment étendre les fonctionnalités principales.

    Points d'extension

    Renderers (Affichage)

    // Interface de base pour tous les renderers
class Renderer {
public:
    virtual void draw(const Grid& grid) = 0;
    virtual void handleEvents() = 0;
    virtual ~Renderer() = default;
};

// Exemple d'implémentation
class CustomRenderer : public Renderer {
    void draw(const Grid& grid) override {
        // Votre logique de rendu personnalisée
    }
    void handleEvents() override {
        // Gestion des événements spécifiques
    }
};

    Règles du jeu

    class Rules {
public:
    virtual bool computeNextState(const Cell& cell, int neighbors) = 0;
    virtual void configure(const Config& config) = 0;
};

// Exemple : Règles personnalisées
class CustomRules : public Rules {
    bool computeNextState(const Cell& cell, int neighbors) override {
        // Vos règles d'évolution personnalisées
        return neighbors == 3 || (cell.isAlive() && neighbors == 2);
    }
};

    Guide d'implémentation

    Créer un nouveau Renderer

    1. Créer une classe héritant de Renderer
    2. Implémenter les méthodes virtuelles pures
    3. Enregistrer dans la factory : 
      RendererFactory::register("custom", []() {
    return std::make_unique<CustomRenderer>();
});

    Ajouter des règles

    1. Hériter de la classe Rules
    2. Implémenter la logique d'évolution
    3. Configurer via fichier : 
      rules:
  type: custom
  params:
    survive: [2,3]
    birth: [3]

    Créer des motifs

    1. Créer un fichier JSON dans /patterns
    2. Structure requise :
    3. Nom et métadonnées
    4. Dimensions
    5. États des cellules
    6. Validation automatique

    Bonnes pratiques

    Pour les Renderers

    ✅ Gérer proprement les ressources ✅ Supporter le redimensionnement ✅ Documentation claire des contrôles ❌ Éviter les dépendances lourdes ❌ Ne pas modifier la logique du jeu

    Pour les Règles

    ✅ Règles simples et documentées ✅ Tests unitaires complets ✅ Configuration flexible ❌ Éviter la complexité inutile ❌ Ne pas modifier l'état global

    Pour les Motifs

    ✅ Format JSON valide ✅ Dimensions raisonnables ✅ Description claire ❌ Éviter les motifs trop grands ❌ Pas de données invalides
