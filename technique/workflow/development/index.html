    Guide de Développement

    Vue d'ensemble

    Philosophie

    Notre workflow est conçu pour être : - Simple à comprendre - Facile à suivre - Efficace en équipe - Maintenable dans le temps

    Outils requis

    Outil Version Usage
    Git 2.34+ Gestion de version
    VS Code/CLion Latest IDE
    CMake 3.20+ Build system
    Clang-format 14+ Style de code

    Workflow de développement

    1. Préparation

    # Mise à jour du dépôt
git checkout main
git pull origin main

# Création de branche
git checkout -b feature/ma-fonctionnalite

    2. Développement

    # Compilation et tests fréquents
mkdir -p build && cd build
cmake ..
make -j$(nproc)
ctest --output-on-failure

# Commits atomiques
git add src/feature.cpp
git commit -m "feat(core): ajouter nouvelle fonctionnalité"

    3. Pull Request

    1. Préparer
    2. Tests passent
    3. Documentation à jour

    4. Code formaté

    5. Soumettre 

      git push origin feature/ma-fonctionnalite
# Créer PR sur GitHub

    Conventions de nommage

    Type Convention Exemple
    Classes PascalCase GameEngine
    Méthodes camelCase updateState()
    Variables camelCase cellCount
    Constantes UPPER_CASE MAX_SIZE

    Messages de commit

    # Format
type(scope): description

# Exemples
feat(grid): ajouter mode torique
fix(ui): corriger affichage statistiques
docs(readme): mettre à jour installation
test(core): ajouter tests unitaires

    GitHub Actions

    name: CI
on: [push, pull_request]

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: Build & Test
        run: |
          mkdir build
          cd build
          cmake ..
          make
          ctest
