    Diagrammes d'États¶

    États du jeu¶

    stateDiagram-v2
    [*] --> Initialize
    Initialize --> Running: start
    Running --> Paused: pause
    Paused --> Running: resume
    Running --> [*]: quit
    Paused --> [*]: quit

    state Running {
        [*] --> UpdateGrid
        UpdateGrid --> RenderFrame
        RenderFrame --> HandleEvents
        HandleEvents --> UpdateGrid
    }

    state Paused {
        [*] --> WaitForInput
        WaitForInput --> HandlePausedEvents
        HandlePausedEvents --> WaitForInput
    }

    États des cellules¶

    stateDiagram-v2
    [*] --> Dead
    Dead --> Alive: birth
    Alive --> Dead: death

    state Alive {
        [*] --> CheckNeighbors
        CheckNeighbors --> Survive: 2-3 neighbors
        CheckNeighbors --> Die: <2 or >3 neighbors
    }

    state Dead {
        [*] --> CheckNeighbors
        CheckNeighbors --> Birth: exactly 3 neighbors
        CheckNeighbors --> StayDead: otherwise
    }

    Flux de données¶

    flowchart TD
    A[Input] --> B{État du jeu}
    B -->|Running| C[Update Grid]
    B -->|Paused| D[Handle Events]
    C --> E[Render]
    D --> E
    E --> B

    Notes d'implémentation¶

    Transitions d'états¶

    • Les transitions sont gérées par le GameManager
    • Chaque état implémente ses propres handlers
    • Les événements sont distribués selon l'état actuel

    Gestion des événements¶

    • Events queue pour les entrées utilisateur
    • Priorité des événements système
    • Callbacks pour les changements d'état

    Persistance¶

    • Sauvegarde automatique de l'état
    • Restauration de la dernière session
    • Points de sauvegarde manuels
