    3. Diagramme de Séquence

    ---
config:
  theme: neutral

sequenceDiagram

  User ->> Programme: Lancer le programme
  Programme ->> Game: Instancie game
  Programme ->> Game: Game.setup()
  Game ->> Config: Config.setup(this)
  Config ->> User: Demande la config
  User -->> Config: Entre la config
  alt User joue a partir d'un fichier
    Config ->> FileHandler: loadGridFromFile()
    FileHandler -->> Config: return vector des 1 et 0
    Config ->> Grid: Grid.initCells()
  end
  alt User joue a partir d'une nouvelle partie
    Config ->> Grid: Grid.initCellsRandom()
  end
  Grid ->> Cell: Creation des cellules
  Config -->>Game: Set les valeurs pour game
  Game ->> IRenderer: Subject.attach()
  Game ->> FileHandler: Subject.attach()


  Programme ->> Game: Game.run()
  loop Pour chaque itération
    Game ->> Grid: Grid.calculateNextGen()
    loop Pour chaque cellule
      Grid ->> IEvolutionStrategy: IEvolutionStrategy.update(Cell) 
    end
    Game ->> Game: Subject.notify()
    Game ->> FileHandler: FileHandler.saveGridToFile()
    Game ->> IRenderer: IRenderer.update()
    IRenderer -->> User: Montre la grille maj
  end

  Game -->> Programme: Terminer après conditions atteintes (stabilité ou itérations max)
  Programme -->> User: Fin
    Illustre le flux d'exécution dans Game::run() : 1. Configuration via Config::setup() 2. Boucle principale dans Game::run() 3. Calcul d'évolution avec Grid::calculateNextGen() 4. Notification des observers via Subject::notify() 5. Rendu et sauvegarde via observers

    4. Diagramme d'Activité

    stateDiagram-v2
    [*] --> Initialisation
    state Initialisation {
        [*] --> LancementProgramme
        LancementProgramme --> MessageBienvenue : "Afficher un message de bienvenue"
        MessageBienvenue --> ChargementOuCreation
        state ChargementOuCreation {
            [*] --> ChargerFichier : "Charger un fichier ?"
            ChargerFichier --> DemanderNomFichier : "Oui"
            DemanderNomFichier --> GrilleDepuisFichier : "Donner le nom du fichier"
            ChargerFichier --> CreationNouvelleGrille : "Non"
            CreationNouvelleGrille --> DemanderTailleGrille : "Donner la taille de la grille"
        }
        GrilleDepuisFichier --> ConfigObstacles
        DemanderTailleGrille --> ConfigObstacles
        ConfigObstacles --> DemanderToroidal : "Ajouter des obstacles ?"
        DemanderToroidal --> DemanderStrategie : "Grille torique ?"
        DemanderStrategie --> DemanderTypeRendu : "Choisir stratégie d'évolution"
        DemanderTypeRendu --> DemanderNbIterations : "Choisir type de rendu (console/graphique)"
        DemanderNbIterations --> DemanderDelay : "Entrer nombre d'itérations"
        DemanderDelay --> ModeAffichage
    }
    state ModeAffichage {
        [*] --> ModeConsole : "Console ou graphique ?"
        ModeConsole --> ConfigTopologie : "Console"
        ModeConsole --> ConfigTopologie : "Graphique"
    }
    ConfigTopologie --> Simulation
    state Simulation {
        [*] --> InitialisationSimulation : "Initialiser Simulation"
        InitialisationSimulation --> VerifierEtatGrille : "gameIsRunning == True ?"

        state VerifierEtatGrille {
            [*] --> ArretSimulation : "Grille stable"
            [*] --> ArretSimulation : "MAX itérations atteint"
            [*] --> IterationSuivante : "Sinon, continuer"
        }

        IterationSuivante --> CalculEtatSuivant : "Calculer état suivant de la grille"
        CalculEtatSuivant --> Affichage : "Afficher la grille"
        Affichage --> VerifierEtatGrille : "Recommencer"
        ArretSimulation --> Fin
    }
    Fin --> [*]

    Représente le workflow implémenté dans main() et Game::run() : 1. Initialisation (Game::setup()) 2. Configuration des paramètres 3. Boucle d'évolution jusqu'à : * Grille stable (Grid::isGridStable()) * Nombre max d'itérations atteint 4. Sauvegarde finale (FileHandler::saveGridToFile())

    Diagrammes Techniques

    Diagramme de Classes

    Classes principales

    • Grid : Gestion de la grille de jeu
    • Cell : Représentation d'une cellule
    • Game : Moteur du jeu
    • Renderer : Interface d'affichage
    • ConsoleRenderer
    • SFMLRenderer

    Stratégies d'évolution

    • ClassicEvolution :
    • Règles standard de Conway
    • Survie avec 2 ou 3 voisins

    • Naissance avec exactement 3 voisins

    • HighLifeEvolution :

    • Variante avec règles étendues
    • Survie avec 2 ou 3 voisins

    • Naissance avec 3 ou 6 voisins

    • Interface IEvolutionStrategy :

    • Méthode evolve() pour règles personnalisées
    • Méthode getName() pour identification
    • Extension facile pour nouvelles règles

    Diagramme de séquence

    Flux principal

    1. Initialisation du jeu
    2. Boucle de simulation
    3. Mise à jour de l'affichage
    4. Gestion des événements
