    Workflow de développement¶

    Gestion des branches¶

    Structure¶

    main
  ├── develop
  │   ├── feature/xxx
  │   ├── fix/xxx
  │   └── docs/xxx
  └── release/x.x.x

    Conventions de nommage¶

    • feature/* : Nouvelles fonctionnalités
    • fix/* : Corrections de bugs
    • docs/* : Documentation
    • release/* : Versions de release
    • hotfix/* : Corrections urgentes

    Cycle de développement¶

    1. Planification¶

    graph LR
    A[Issue] --> B[Discussion]
    B --> C[Planning]
    C --> D[Assignment]

    2. Développement¶

    # Création de branche
git checkout develop
git pull origin develop
git checkout -b feature/ma-feature

# Développement itératif
git add .
git commit -m "feat: description"
git push origin feature/ma-feature

    3. Revue de code¶

    graph TD
    A[PR Created] --> B[Automated Checks]
    B --> C[Code Review]
    C --> D[Changes Requested]
    D --> B
    C --> E[Approved]
    E --> F[Merge]

    Versionnement¶

    Semantic Versioning¶

    MAJOR.MINOR.PATCH
  │     │     └─ Corrections de bugs
  │     └─────── Nouvelles fonctionnalités rétrocompatibles
  └───────────── Changements incompatibles

    Changelog¶

    # Changelog

## [1.2.0] - 2024-03-15
### Added
- Nouvelle fonctionnalité X
- Support pour Y

### Fixed
- Correction du bug Z
- Amélioration des performances

### Changed
- Mise à jour de la dépendance A

    CI/CD¶

    Pipeline¶

    stages:
  - build
  - test
  - analyze
  - deploy

build:
  stage: build
  script:
    - cmake .
    - make

test:
  stage: test
  script:
    - ./run_tests.sh

analyze:
  stage: analyze
  script:
    - ./static_analysis.sh

    Release¶

    Processus¶

    1. Création branche release
    2. Tests finaux
    3. Mise à jour version
    4. Merge dans main
    5. Tag version
    6. Déploiement

    Commandes¶

    # Création release
git checkout -b release/1.2.0
./scripts/update_version.sh 1.2.0
git commit -am "chore: bump version to 1.2.0"

# Finalisation
git checkout main
git merge release/1.2.0
git tag -a v1.2.0 -m "Version 1.2.0"
git push origin main --tags
